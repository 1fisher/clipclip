## Findings (non-obvious facts with file:line refs)
- pbxproj used PBXFileSystemSynchronizedRootGroup — new files auto-detected by Xcode
- SwiftData circular @Relationship causes macro expansion error — use @Relationship only on one side
- Double.formatted() is a method (returns String) not a property — used .formattedDuration extension
- iOS 26.5 SDK available but 26.5 simulator runtime not installed — can't run on simulator, only device build
- @Observable macro + @MainActor isolation causes warnings for static method calls from non-isolated contexts
- .duration on AVAsset deprecated in iOS 16 — use load(.duration) async

## Learnings (patterns / pitfalls for next turn)
- VideoComposer uses VideoResolver closure pattern for flexible asset loading
- PhotosPickerItem.loadTransferable(type: Data.self) for importing video data
- .photosPicker() modifier for SwiftUI-native photo picking
- Video files stored in Documents/ImportedVideos/ as UUID.mp4
